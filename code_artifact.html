TDC 기반 동역학 제어

로봇 팔의 정확한 관성행렬을 알 수 없는 한계를 극복하기 위해 Time Delay Control(TDC)을 적용했습니다. 이전 시점의 동역학 항으로 현 시점을 추정하여 비선형성을 상쇄합니다.

\(\tau_{t}^{TDC} = -\hat{M}\Phi_{t} + \hat{M}(\ddot{q}_{d} + K_{d}\dot{e}_{t} + K_{p}e_{t})\)

MATLAB 시뮬레이션 결과, PID 대비 정밀한 궤적 추종 입증

SLAM & Path Planning

Turtlebot의 `amcl` 패키지를 활용하여 환경 지도를 작성하고 위치를 추정합니다. Global Planner와 Local Planner를 결합하여 안정적인 자율 주행을 달성했습니다.

  • • A* Algorithm: 휴리스틱(추정비용)을 통한 최적의 전역 경로 설정
  • • DWA: 속도/가속도 범위를 고려한 실시간 동적 장애물 회피

YOLOv5 실시간 객체 인식

연산량이 제한된 라즈베리파이(Raspberry Pi 4) 환경에 맞추어, 빠르고 가벼운 단일 CNN 네트워크인 YOLOv5 모델을 채택하였습니다.

목표 객체를 인식하고, 좌표 데이터를 ROS 메세지로 Publish하여 모바일 플랫폼의 접근 및 로봇 팔의 파지 동작을 트리거합니다.

Engineering Challenges & Troubleshooting

실제 하드웨어 통합 과정에서 발생한 문제점 분석 및 최적화 과정

1 SLAM 맵 매칭 오류 및 하드웨어 간섭 노이즈 해결

기본 Gmapping 파라미터 사용 시 로봇 팔(Manipulator)이 LiDAR 센서를 가려 자신의 팔을 장애물로 인식하거나 맵이 뭉개지는 현상이 발생했습니다.

LiDAR 데이터 최적화

  • • FOV 제한: 후방 노이즈 제거를 위해 센서 수신을 전방 180도(-1.57 ~ +1.57 rad)로 제한
  • • 거리 필터링: 로봇 팔 간섭 방지용 20cm 이내 데이터 무시 및 최대 인식 거리 3.0m로 하향 조정

Gmapping 파라미터 튜닝

  • • minimumScore: 실내 복잡도에 맞춰 140 → 50으로 하향
  • • particles: 위치 추정 정확도 향상을 위해 100 → 150으로 증가
  • • linearUpdate: 맵 정밀도를 위해 1.0m → 0.2m로 세밀화

2 Local Path Planning (DWA) 최적화

생성된 전역 경로(Global Path)를 기반으로 이동할 때, 동적 장애물을 실시간으로 회피하기 위해 DWA 플래너를 최적화했습니다. 차량의 현재 속도에서 제동 가능한 속도 범위 내의 선속도(\(v\))와 각속도(\(w\)) 조합 중, 목적 함수 \(G(v, w)\)가 최대가 되는 회피 궤적을 추종합니다.

\[ G(v,w) = \alpha \times \text{heading}(v,w) + \beta \times \text{clearance}(v,w) + \gamma \times \text{velocity}(v,w) \]

3 Vision AI 한계 극복 및 경량화 (OpenCV → YOLOv5 + Edge TPU)

초기 OpenCV를 활용한 Rule-based 방식(HSV 필터링 등)은 조명 변화에 취약하여 복잡한 배경에서 오인식이 다수 발생했습니다. 이를 극복하기 위해 딥러닝 모델을 도입하고 엣지 디바이스 환경에 맞게 최적화했습니다.

하드웨어 가속기 적용

라즈베리파이(Raspberry Pi)의 부족한 연산 능력을 극복하고 실시간 인퍼런스(Inference)를 달성하기 위해 Google Coral Edge TPU를 도입했습니다.

모델 양자화 (Quantization)

학습된 PyTorch 모델을 ONNX를 거쳐 TFLite 포맷으로 변환하고, INT8 양자화를 적용하여 모델 크기를 줄이고 Edge TPU 컴파일러로 최적화(`_edgetpu.tflite`)했습니다.

성과 및 향후 과제

System Integration (시스템 통합)

MATLAB을 Master로 구성하여 각 파트(주행, 조작, 인식)의 간섭을 제어했습니다. 조작 시에는 주행을 멈추고, 주행 시에는 조작을 멈추는 ROS 기반 Master-Slave 구조를 성공적으로 구축하여 전체 자율화 시나리오를 완성했습니다.

Limitation (한계점)

모바일 환경 특성상 배터리 전원에 의존하게 되는데, 주행과 매니퓰레이터 조작을 병행할 시 전압 강하로 인한 로봇 팔 제어 불안정 현상이 관찰되었습니다. 향후 독립적인 BMS 구성이나 하드웨어 전력 분배 최적화를 통한 개선이 필요합니다.
